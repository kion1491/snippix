# 간단한 웹 이미지 편집기 — 기능 정의서

## 1. 프로덕트 개요

**한 줄 정의**
포토샵의 복잡함 없이, 개발자와 초보자가 자주 쓰는 이미지 작업만 빠르게 처리하는 웹 기반 이미지 편집기.

**타겟 사용자**
- 블로그/SNS 이미지 가공이 필요한 비개발자
- 썸네일/아이콘/스프라이트 등 빠른 가공이 필요한 개발자
- 가입·설치 없이 즉시 쓰고 싶은 사용자

**핵심 가치**
- 가입 불필요, 즉시 사용
- 클라이언트 사이드 처리 (서버에 이미지 업로드 X = 프라이버시 보장)
- 한 화면에서 업로드 → 편집 → 다운로드 완결

## 2. MUST HAVE (MVP 핵심 기능)

### 2.1 파일 입력
- 드래그 & 드롭 업로드
- 파일 선택 다이얼로그
- 클립보드 붙여넣기 (Ctrl+V / Cmd+V)
- URL로 이미지 불러오기
- 다중 파일 업로드 (배치 처리용)
- 지원 포맷: JPG, JPEG, PNG, WebP, GIF(첫 프레임), SVG, AVIF

### 2.2 자르기 (Crop)
- 자유 비율 자르기
- 비율 프리셋: 1:1, 4:3, 3:4, 16:9, 9:16, 21:9
- 픽셀 단위 정확한 영역 입력 (W × H + X, Y 좌표)
- 소셜미디어 프리셋
  - Instagram 정사각형 (1080×1080)
  - Instagram 스토리/릴스 (1080×1920)
  - YouTube 썸네일 (1280×720)
  - 네이버 블로그 권장 가로폭 (966px)
  - Facebook 커버, X 헤더 등
- 자르기 영역을 핸들로 직관적 조정

### 2.3 슬라이스 (Slice / 분할)
- 격자 분할: N × M 그리드 입력 (예: 3×3, 2×4)
- 균등 분할 (가로/세로 픽셀 또는 개수 기준)
- 미리보기로 분할선 시각화
- 슬라이스된 조각을 ZIP으로 일괄 다운로드
- 파일명 자동 명명: `{원본명}_r{행}_c{열}.{확장자}`

### 2.4 크기 조정 (Resize)
- 픽셀 입력 (W × H)
- 퍼센트 입력 (예: 50%)
- 가로세로 비율 잠금 토글
- 한쪽 변 기준 자동 계산 (긴 변 / 짧은 변 기준)
- 업스케일 시 경고 표시

### 2.5 용량 줄이기 (Compress)
- 품질 슬라이더 (1–100)
- 목표 파일 크기 모드 (예: "500KB 이하로 자동 압축")
- 포맷 변환 동시 적용 (PNG → JPG / WebP 변환으로 추가 용량 절감)
- 압축 전/후 용량 및 화질 비교 미리보기 (Before/After 슬라이더)
- 배치 압축 지원

### 2.6 포맷 변환
- 변환 매트릭스: PNG ↔ JPG ↔ WebP ↔ AVIF
- 투명 배경 처리 옵션 (PNG → JPG 시 배경색 선택)
- 각 포맷별 품질 옵션

### 2.7 기본 회전/반전
- 90° 단위 회전 (좌/우)
- 자유 회전 (각도 입력)
- 좌우 반전, 상하 반전

### 2.8 작업 흐름
- Undo / Redo (Ctrl+Z / Ctrl+Shift+Z)
- 원본 보존 (언제든 "원본으로 리셋" 가능)
- 작업 전/후 비교 뷰

### 2.9 내보내기 (Export)
- 단일 파일 다운로드
- 다중 파일 ZIP 다운로드
- 파일명 규칙 사용자 지정 (prefix, suffix, 자동 번호)
- 최종 포맷/품질 선택

## 3. SHOULD HAVE (2차 우선순위)

- **일괄 처리 모드**: 여러 이미지에 동일 작업(리사이즈 + 압축 + 포맷 변환)을 한 번에 적용
- **메타데이터 제거** (EXIF) — 프라이버시 옵션
- **단축키 시스템** (C = Crop, R = Resize, S = Slice 등)
- **작업 히스토리 패널** (단계별 되돌리기)
- **모바일 반응형 UI**

---

## 4. WON'T HAVE (명시적으로 배제)

> ⚠️ **이 섹션은 반드시 Claude Code에 함께 전달할 것.** 안 그러면 친절하게 다 만들어줘서 결국 포토샵 짝퉁이 됨.

다음 기능은 만들지 않음:
- 레이어 시스템
- 필터/효과 (블러, 샤픈, 빈티지, 색상 보정 등)
- 브러시/그리기 도구
- 텍스트 추가
- 셰이프/도형 추가
- 마스킹, 채널, 곡선 등 색 보정
- 회원가입/계정 시스템
- 클라우드 저장
- 협업 기능
- AI 이미지 생성/배경 제거

이런 기능이 필요한 사용자는 Photoshop / Figma / Photopea / remove.bg 등 다른 도구를 사용한다.

## 5. 비기능 요구사항

### 5.1 성능
- **모든 이미지 처리는 클라이언트 사이드(브라우저 내)** 에서 수행
- 서버 업로드 절대 금지
- 10MB 이미지 기준, 주요 작업 3초 이내 완료
- 대용량 이미지(50MB+) 처리 시 UI 멈춤 방지를 위해 **Web Worker 활용** 권장

### 5.2 프라이버시
- "이미지가 서버로 전송되지 않습니다"를 UI 하단에 명시
- 페이지 새로고침 시 데이터 자동 소멸 (의도된 동작이며 사용자에게 안내)

### 5.3 UX 원칙
- 가입/로그인 없이 첫 화면에서 즉시 작업 시작
- 도구 전환 시 페이지 이동 없음 (단일 화면 완결)
- 모든 작업 결과는 실시간 미리보기
- 에러 메시지는 친절한 한국어 (예: "10MB 이하 이미지만 권장합니다")
- 최초 진입 시 빈 화면이 아닌 "여기에 이미지를 끌어다 놓으세요" 같은 명확한 가이드

### 5.4 접근성
- 키보드만으로 주요 작업 가능
- 모바일/태블릿 터치 제스처 지원 (핀치 줌 등)

│                                  │              │

## 7. 개발 우선순위 (권장 빌드 순서)

Claude Code에 단계적으로 시킬 것을 권장:

1. **1단계**: 파일 입력(2.1) + 캔버스 표시 + 다운로드(2.9 단일)
2. **2단계**: 자르기(2.2) + Undo/Redo(2.8)
3. **3단계**: 크기 조정(2.4) + 용량 줄이기(2.5) + 포맷 변환(2.6)
4. **4단계**: 회전/반전(2.7)
5. **5단계**: 슬라이스(2.3) + ZIP 다운로드
6. **6단계**: SHOULD HAVE 항목
